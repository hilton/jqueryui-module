Dialog

Use a dialog (i.e. a dialogue box) to show additional user-interface elements that are not part of the main page. A common use case is to show a ‘selector pop-up’ that provides a richer user-interface for selecting a value than a pick-list. This is especially useful when the dialog contents are lazy-loaded.

Client

The dialog.js JavaScript attaches a click handler the the button in the following view mark-up. This click handler uses the jQuery UI dialog plug-in to open a dialog box, and loads the dialog contents from the resource given by the data-url attribute value, via an Ajax request.

<input name="location">
<button data-url="@{jqueryui.Dialog.timeZones()}">Select Time Zone</button>

Server

The server-side resource that is used to populate the dialog contents is simply a server-side view. A more realistic example might include server-side filtering of a long list, so that the selector can be used to select one of thousands of values.
